Interview Questions & Answers
EasyWhat is the two-pointer technique? When do you use it?
▼

Best Answer

Two pointers is a technique where you maintain two indices (usually left and right) that move towards each other or in the same direction to solve problems in O(n) instead of O(n²).

Use it when:

  • Finding pairs in a sorted array that sum to a target
  • Checking if a string is a palindrome
  • Removing duplicates from a sorted array
  • Container with most water problem

Example Code

// Two Sum in sorted array
function twoSum(arr, target) {
  let left = 0, right = arr.length - 1;
  while (left < right) {
    const sum = arr[left] + arr[right];
    if (sum === target) return [left, right];
    else if (sum < target) left++;
    else right--;
  }
  return [-1, -1];
}
💡 Interview Tip

Always mention you're reducing time from O(n²) to O(n) and explain WHY the pointers move inward — this shows deep understanding.

MediumExplain the Sliding Window technique with an example.
▼

Best Answer

Sliding Window maintains a window of elements and slides it across the array/string, expanding or contracting as needed. It's perfect for subarray/substring problems.

Two types:

  • Fixed size: window size k stays constant — find max sum of k elements
  • Variable size: window shrinks/grows — longest substring without repeating characters

Classic Example — Longest Substring Without Repeats

function lengthOfLongestSubstring(s) {
  const seen = new Map();
  let left = 0, maxLen = 0;
  for (let right = 0; right < s.length; right++) {
    if (seen.has(s[right]) && seen.get(s[right]) >= left) {
      left = seen.get(s[right]) + 1;
    }
    seen.set(s[right], right);
    maxLen = Math.max(maxLen, right - left + 1);
  }
  return maxLen;
}
💡 Interview Tip

Draw the window on a whiteboard/paper and show how it moves — interviewers love visual explanations. Mention O(n) time over O(n²) brute force.

EasyWhat is Kadane's Algorithm? Find maximum subarray sum.
▼

Best Answer

Kadane's algorithm finds the maximum sum contiguous subarray in O(n) time. The key insight: at each position, decide whether to extend the current subarray or start fresh.

Code

function maxSubArray(nums) {
  let maxSoFar = nums[0];
  let currentMax = nums[0];
  for (let i = 1; i < nums.length; i++) {
    currentMax = Math.max(nums[i], currentMax + nums[i]);
    maxSoFar = Math.max(maxSoFar, currentMax);
  }
  return maxSoFar;
}
// [-2,1,-3,4,-1,2,1,-5,4] → 6 ([4,-1,2,1])
💡 Interview Tip

Trace through a small example step by step. Explain the core decision: "If adding the current element makes the sum worse than starting fresh, restart."

MediumHow do you find all duplicates in an array in O(n) time and O(1) space?
▼

Best Answer

If the array contains values in range [1, n], use index-marking — negate the value at index arr[i]-1. If you find a negative value there again, it's a duplicate.

Code

function findDuplicates(nums) {
  const result = [];
  for (let i = 0; i < nums.length; i++) {
    const idx = Math.abs(nums[i]) - 1;
    if (nums[idx] < 0) result.push(Math.abs(nums[i]));
    else nums[idx] = -nums[idx];
  }
  return result;
}
💡 Interview Tip

This only works when values are in [1, n]. Always state this constraint clearly. Interviewers want to see you identify assumptions.

HardTrapping Rain Water — explain your approach.
▼

Best Answer — Two Pointer O(n) O(1)

Water trapped at index i = min(maxLeft, maxRight) - height[i]. Use two pointers instead of precomputed arrays.

Code

function trap(height) {
  let left = 0, right = height.length - 1;
  let leftMax = 0, rightMax = 0, water = 0;
  while (left < right) {
    if (height[left] <= height[right]) {
      leftMax = Math.max(leftMax, height[left]);
      water += leftMax - height[left];
      left++;
    } else {
      rightMax = Math.max(rightMax, height[right]);
      water += rightMax - height[right];
      right--;
    }
  }
  return water;
}
💡 Interview Tip

Start with brute force O(n²), then optimize to O(n) with prefix arrays, then impress with the O(1) space two-pointer solution. Show your thought process evolving.

EasyHow do you reverse a linked list?
▼

Best Answer — Iterative (O(n) time, O(1) space)

function reverseList(head) {
  let prev = null, curr = head;
  while (curr) {
    const next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
  }
  return prev; // new head
}

Recursive Approach (O(n) space)

function reverseRecursive(head) {
  if (!head || !head.next) return head;
  const newHead = reverseRecursive(head.next);
  head.next.next = head;
  head.next = null;
  return newHead;
}
💡 Interview Tip

Interviewers love seeing BOTH iterative and recursive. Mention iterative is preferred in production (no stack overflow). Trace through 3 nodes manually.

MediumDetect a cycle in a linked list — Floyd's Algorithm.
▼

Best Answer

Floyd's Cycle Detection uses slow (1 step) and fast (2 steps) pointers. If they meet, there's a cycle. To find the cycle start, reset one pointer to head and move both one step at a time.

Code

function detectCycle(head) {
  let slow = head, fast = head;
  // Phase 1: detect cycle
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) {
      // Phase 2: find start
      slow = head;
      while (slow !== fast) {
        slow = slow.next;
        fast = fast.next;
      }
      return slow; // cycle start node
    }
  }
  return null;
}
💡 Interview Tip

Explain the mathematical proof WHY this works — distance from head to cycle start equals distance from meeting point to cycle start. This shows you truly understand the algorithm.

EasyFind the middle of a linked list.
▼

Best Answer — Slow/Fast Pointer

function middleNode(head) {
  let slow = head, fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
  }
  return slow; // when fast reaches end, slow is at middle
}
💡 Interview Tip

This is the foundation for many problems (palindrome list, sort list). Mention it's O(n) and only requires one pass — no length calculation needed.

HardMerge K sorted linked lists — what's the optimal approach?
▼

Best Answer — Min Heap O(N log K)

Push heads of all K lists into a min-heap. Repeatedly extract minimum, add to result, push that node's next. This is O(N log K) where N = total nodes, K = number of lists.

Alternative — Divide & Conquer O(N log K)

Merge lists pairwise recursively, like merge sort. Also O(N log K) but doesn't need a heap data structure.

💡 Interview Tip

Start with naive O(KN) approach, then optimize. Always discuss tradeoffs. Min-heap is more intuitive; divide & conquer is elegant. State both and let interviewer pick.

EasyCheck if parentheses are balanced using a stack.
▼

Best Answer

function isValid(s) {
  const stack = [];
  const map = { ')': '(', '}': '{', ']': '[' };
  for (const ch of s) {
    if ('({['.includes(ch)) stack.push(ch);
    else if (stack.pop() !== map[ch]) return false;
  }
  return stack.length === 0;
}
💡 Interview Tip

Extend your answer: "This can be generalized to any matching pair problem like HTML tags or XML validation."

MediumWhat is a Monotonic Stack? Give a problem example.
▼

Best Answer

A monotonic stack maintains elements in increasing or decreasing order. When a new element breaks the order, pop until the order is restored. Used for "Next Greater Element" type problems.

Next Greater Element

function nextGreaterElement(nums) {
  const result = new Array(nums.length).fill(-1);
  const stack = []; // indices, monotonically decreasing values
  for (let i = 0; i < nums.length; i++) {
    while (stack.length && nums[i] > nums[stack[stack.length - 1]]) {
      const idx = stack.pop();
      result[idx] = nums[i];
    }
    stack.push(i);
  }
  return result;
}
💡 Interview Tip

Monotonic stacks solve: Next Greater/Smaller Element, Largest Rectangle in Histogram, Daily Temperatures — all in O(n). This is a must-know pattern.

HardDesign an LRU Cache with O(1) get and put.
▼

Best Answer — HashMap + Doubly Linked List

Use a HashMap for O(1) lookup and a doubly linked list to track recency. Most recently used = head, least recently used = tail. On access, move node to head. On eviction, remove tail.

Key Insight

  • HashMap: key → node (O(1) access)
  • DLL: maintains order (O(1) insert/delete)
  • On get: move node to front
  • On put over capacity: remove tail node + remove from map
💡 Interview Tip

Draw the data structure first. Explain each operation before coding. LRU Cache is asked at Google, Amazon, Microsoft. In Python, use OrderedDict for an elegant solution.

Interview Questions & Answers
EasyWhat are the tree traversals and when do you use each?
▼

Best Answer

  • Inorder (L→Root→R): BST inorder gives sorted output — use for sorted operations
  • Preorder (Root→L→R): Copy/serialize a tree structure
  • Postorder (L→R→Root): Delete a tree, evaluate expression trees
  • Level Order (BFS): Find shortest path, print level by level

Code — All 4 Traversals

// Recursive inorder
function inorder(root, result = []) {
  if (!root) return result;
  inorder(root.left, result);
  result.push(root.val);
  inorder(root.right, result);
  return result;
}
// Level order (BFS)
function levelOrder(root) {
  if (!root) return [];
  const queue = [root], result = [];
  while (queue.length) {
    const level = [];
    const size = queue.length;
    for (let i = 0; i < size; i++) {
      const node = queue.shift();
      level.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }
    result.push(level);
  }
  return result;
}
💡 Interview Tip

Memorize: Inorder of BST = sorted array. This shortcut solves many BST problems instantly.

EasyFind the height/depth of a binary tree.
▼

Best Answer

function maxDepth(root) {
  if (!root) return 0;
  return 1 + Math.max(maxDepth(root.left), maxDepth(root.right));
}
💡 Interview Tip

This is a postorder traversal. The recursion returns "what's the max depth below me?" — classic bottom-up tree thinking. Time: O(n), Space: O(h) where h = height.

MediumValidate if a Binary Tree is a valid BST.
▼

Best Answer — Pass min/max bounds

A common mistake: only comparing node with direct children. The correct approach passes valid range (min, max) to every node.

Code

function isValidBST(root, min = -Infinity, max = Infinity) {
  if (!root) return true;
  if (root.val <= min || root.val >= max) return false;
  return isValidBST(root.left, min, root.val) &&
         isValidBST(root.right, root.val, max);
}
💡 Interview Tip

Show the interviewer the common wrong answer first (only comparing parent-child), then explain why it fails with a counterexample, then give the correct solution. This shows sophisticated thinking.

MediumFind Lowest Common Ancestor (LCA) of two nodes.
▼

Best Answer

For a general binary tree: recursively search left and right. If both sides return non-null, current node is LCA. For BST: if both targets are smaller, go left; if both larger, go right; else current is LCA.

Code — General Binary Tree

function lowestCommonAncestor(root, p, q) {
  if (!root || root === p || root === q) return root;
  const left = lowestCommonAncestor(root.left, p, q);
  const right = lowestCommonAncestor(root.right, p, q);
  return left && right ? root : left || right;
}
💡 Interview Tip

This is a classic problem. The elegant 3-line solution impresses interviewers. Always handle the BST variant separately — it's more efficient O(log n) vs O(n).

HardFind the diameter of a binary tree.
▼

Best Answer

Diameter = longest path between any two nodes. At each node, it could pass through that node = left_height + right_height. Track the global max.

Code

function diameterOfBinaryTree(root) {
  let maxDiam = 0;
  function height(node) {
    if (!node) return 0;
    const left = height(node.left);
    const right = height(node.right);
    maxDiam = Math.max(maxDiam, left + right);
    return 1 + Math.max(left, right);
  }
  height(root);
  return maxDiam;
}
💡 Interview Tip

The key insight: the diameter at any node = left subtree height + right subtree height. Many tree "global maximum" problems use this pattern of computing local + updating global.

EasyHow does a HashMap work internally? What is collision handling?
▼

Best Answer

A HashMap uses a hash function to map keys to array indices (buckets). When two keys hash to the same index, that's a collision.

Collision Handling Methods:

  • Chaining: Each bucket holds a linked list of entries (Java's HashMap uses this)
  • Open Addressing: Linear probing — find next empty slot
  • Double Hashing: Use second hash function to find next slot

Load Factor

When load factor (entries/buckets) exceeds threshold (~0.75), the map resizes and rehashes — O(n) amortized, so average insert is still O(1).

💡 Interview Tip

This is a favorite system design warm-up. Mention Java's HashMap uses chaining + converts to red-black tree at 8+ entries in a bucket (Java 8+). Shows you know internals.

EasyTwo Sum — find pair that adds to target.
▼

Best Answer — HashMap O(n)

function twoSum(nums, target) {
  const seen = new Map(); // value → index
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (seen.has(complement)) {
      return [seen.get(complement), i];
    }
    seen.set(nums[i], i);
  }
}
💡 Interview Tip

Always start: "Brute force is O(n²). We can optimize to O(n) using a HashMap by storing complements." Shows problem-solving progression.

MediumGroup anagrams together from a list of strings.
▼

Best Answer

function groupAnagrams(strs) {
  const map = new Map();
  for (const str of strs) {
    const key = str.split('').sort().join(''); // sorted chars as key
    if (!map.has(key)) map.set(key, []);
    map.get(key).push(str);
  }
  return [...map.values()];
}
// O(n * k log k) where k = max string length
💡 Interview Tip

Mention optimization: instead of sorting (k log k), use char frequency array of 26 as key → O(n*k) total. Interviewers may ask for this follow-up.

Interview Questions & Answers
EasyBinary Search — implement it and explain the edge cases.
▼

Best Answer

function binarySearch(arr, target) {
  let left = 0, right = arr.length - 1;
  while (left <= right) {       // ← <= not <
    const mid = left + Math.floor((right - left) / 2); // ← avoids overflow
    if (arr[mid] === target) return mid;
    else if (arr[mid] < target) left = mid + 1;
    else right = mid - 1;
  }
  return -1;
}

Common Edge Cases:

  • Empty array → return -1
  • One element → check it
  • Overflow: use left + (right-left)/2 not (left+right)/2
  • Infinite loop: ensure left/right converge (mid+1 and mid-1)
💡 Interview Tip

Explain the overflow fix — this shows C++/Java awareness. Also mention variants: find first occurrence, last occurrence, search in rotated array.

MediumSearch in a rotated sorted array.
▼

Best Answer — Modified Binary Search O(log n)

function search(nums, target) {
  let left = 0, right = nums.length - 1;
  while (left <= right) {
    const mid = left + Math.floor((right - left) / 2);
    if (nums[mid] === target) return mid;
    // Left half is sorted
    if (nums[left] <= nums[mid]) {
      if (target >= nums[left] && target < nums[mid]) right = mid - 1;
      else left = mid + 1;
    } else { // Right half is sorted
      if (target > nums[mid] && target <= nums[right]) left = mid + 1;
      else right = mid - 1;
    }
  }
  return -1;
}
💡 Interview Tip

Key insight: at least one half is always sorted after rotation. Identify which half, check if target is in the sorted half, then binary search accordingly.

EasyBFS vs DFS — when do you use each?
▼

Best Answer

  • BFS (Queue): Shortest path in unweighted graph, level-order traversal, "closest/minimum hops"
  • DFS (Stack/Recursion): Cycle detection, topological sort, connected components, backtracking

BFS Template

function bfs(graph, start) {
  const visited = new Set([start]);
  const queue = [start];
  while (queue.length) {
    const node = queue.shift();
    for (const neighbor of graph[node]) {
      if (!visited.has(neighbor)) {
        visited.add(neighbor);
        queue.push(neighbor);
      }
    }
  }
}
💡 Interview Tip

Rule of thumb: "Shortest path → BFS. Explore all paths / detect cycles → DFS." Most grid problems (islands, shortest path) use BFS for shortest path.

MediumNumber of Islands — count connected components.
▼

Best Answer — DFS O(m×n)

function numIslands(grid) {
  let count = 0;
  for (let r = 0; r < grid.length; r++) {
    for (let c = 0; c < grid[0].length; c++) {
      if (grid[r][c] === '1') {
        count++;
        dfs(grid, r, c);
      }
    }
  }
  return count;
}
function dfs(grid, r, c) {
  if (r < 0 || r >= grid.length || c < 0 || c >= grid[0].length
      || grid[r][c] !== '1') return;
  grid[r][c] = '0'; // mark visited
  dfs(grid, r+1, c); dfs(grid, r-1, c);
  dfs(grid, r, c+1); dfs(grid, r, c-1);
}
💡 Interview Tip

This pattern (flood fill DFS/BFS) applies to: Max Area of Island, Pacific Atlantic Water Flow, Surrounded Regions. Mention Union-Find as an alternative for follow-up.

MediumWhat is Topological Sort? When is it used?
▼

Best Answer

Topological sort orders nodes in a DAG such that all dependencies come before dependents. Used for task scheduling, build systems, course prerequisites.

Kahn's Algorithm (BFS-based)

function topologicalSort(numNodes, edges) {
  const inDegree = new Array(numNodes).fill(0);
  const graph = Array.from({length: numNodes}, () => []);
  for (const [u, v] of edges) {
    graph[u].push(v);
    inDegree[v]++;
  }
  const queue = [];
  for (let i = 0; i < numNodes; i++)
    if (inDegree[i] === 0) queue.push(i);
  const result = [];
  while (queue.length) {
    const node = queue.shift();
    result.push(node);
    for (const next of graph[node]) {
      inDegree[next]--;
      if (inDegree[next] === 0) queue.push(next);
    }
  }
  return result.length === numNodes ? result : []; // empty = cycle
}
💡 Interview Tip

If result.length !== numNodes, there's a cycle → impossible ordering. This detects cycles in directed graphs too. Classic: "Course Schedule" on LeetCode.

HardExplain Dijkstra's algorithm — when does it fail?
▼

Best Answer

Dijkstra finds shortest path from source to all nodes in a weighted graph using a min-heap. Greedily picks the unvisited node with minimum distance.

When it fails:

  • Negative edge weights — use Bellman-Ford instead
  • Negative cycles — no valid shortest path exists

Complexity: O((V+E) log V) with priority queue

💡 Interview Tip

Dijkstra = greedy + BFS + min-heap. Always mention the negative weight limitation. Follow-up: "For negative weights use Bellman-Ford O(VE); for all-pairs use Floyd-Warshall O(V³)."

  5. Interval DP — palindromic substrings, burst balloons

📌 Memoization vs Tabulation

  • Memoization (Top-Down): Recursion + cache. Natural to think about, may hit stack limit
  • Tabulation (Bottom-Up): Iterative, no recursion overhead, harder to think about
  • Space Optimization: Many 2D DP → 1D by noting you only need previous row
Interview Questions & Answers
EasyWhat is Dynamic Programming? How do you identify a DP problem?
▼

Best Answer

DP solves problems by breaking them into overlapping subproblems, solving each once, and storing results. Two necessary conditions:

  • Optimal Substructure: Optimal solution uses optimal solutions of subproblems
  • Overlapping Subproblems: Same subproblems are solved multiple times

How to identify DP:

  • Problem asks for max/min/count/true-false
  • You can make choices at each step
  • Keywords: "number of ways", "minimum cost", "longest", "can you reach"
💡 Interview Tip

Say: "I notice this has optimal substructure and overlapping subproblems, so DP applies. Let me define the state and transition." This structure impresses interviewers.

EasyFibonacci — memoization vs tabulation.
▼

Memoization (Top-Down)

function fib(n, memo = {}) {
  if (n <= 1) return n;
  if (memo[n]) return memo[n];
  return memo[n] = fib(n-1, memo) + fib(n-2, memo);
}

Tabulation (Bottom-Up) — Space Optimized O(1)

function fib(n) {
  if (n <= 1) return n;
  let prev2 = 0, prev1 = 1;
  for (let i = 2; i <= n; i++) {
    [prev2, prev1] = [prev1, prev1 + prev2];
  }
  return prev1;
}
💡 Interview Tip

Always show the space optimization. Going from O(n) space to O(1) by keeping only last 2 values demonstrates mastery. Apply this to Climbing Stairs, House Robber too.

Medium0/1 Knapsack — items with weights and values, max value within capacity.
▼

Best Answer — O(n×W) time and space

function knapsack(weights, values, W) {
  const n = weights.length;
  // dp[i][w] = max value using first i items with capacity w
  const dp = Array.from({length: n+1}, () => new Array(W+1).fill(0));
  for (let i = 1; i <= n; i++) {
    for (let w = 0; w <= W; w++) {
      dp[i][w] = dp[i-1][w]; // don't include item i
      if (weights[i-1] <= w) {
        dp[i][w] = Math.max(dp[i][w],
          dp[i-1][w - weights[i-1]] + values[i-1]);
      }
    }
  }
  return dp[n][W];
}

Space Optimized — O(W)

// Traverse w from right to left to avoid using item twice
for (let w = W; w >= weights[i-1]; w--)
  dp[w] = Math.max(dp[w], dp[w - weights[i-1]] + values[i-1]);
💡 Interview Tip

State definition is everything. Say: "dp[i][w] = max value using first i items with capacity w." Then derive the recurrence naturally from that definition.

MediumLongest Common Subsequence (LCS).
▼

Best Answer

function lcs(text1, text2) {
  const m = text1.length, n = text2.length;
  const dp = Array.from({length: m+1}, () => new Array(n+1).fill(0));
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (text1[i-1] === text2[j-1])
        dp[i][j] = dp[i-1][j-1] + 1;
      else
        dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
    }
  }
  return dp[m][n];
} // O(m*n) time and space
💡 Interview Tip

LCS is the foundation of: Longest Common Substring, Edit Distance, Shortest Common Supersequence, Delete Operations. Know LCS and you unlock all of them.

HardCoin Change — minimum coins to make amount.
▼

Best Answer — Unbounded Knapsack

function coinChange(coins, amount) {
  const dp = new Array(amount + 1).fill(Infinity);
  dp[0] = 0;
  for (let i = 1; i <= amount; i++) {
    for (const coin of coins) {
      if (coin <= i) {
        dp[i] = Math.min(dp[i], dp[i - coin] + 1);
      }
    }
  }
  return dp[amount] === Infinity ? -1 : dp[amount];
}
💡 Interview Tip

Greedy fails here (try coins [1,3,4] for amount 6 — greedy gives 4+1+1=3 coins, DP gives 3+3=2). Always explain WHY greedy fails before jumping to DP.

SQL & Database Questions

From basic SELECT to window functions, indexes, and system design — complete database interview guide.

📌 SQL Execution Order

  1. FROM — load tables
  2. JOIN — combine
  3. WHERE — filter rows
  4. GROUP BY — group
  5. HAVING — filter groups
  6. SELECT — columns
  7. ORDER BY — sort
  8. LIMIT — restrict rows

📌 JOIN Types

  • INNER JOIN — matching rows in both
  • LEFT JOIN — all from left, matching from right
  • RIGHT JOIN — all from right, matching from left
  • FULL OUTER JOIN — all rows from both
  • CROSS JOIN — Cartesian product
  • SELF JOIN — table joined with itself
Interview Questions & Answers
EasyWhat is the difference between WHERE and HAVING?
▼

Best Answer

  • WHERE filters rows BEFORE grouping — cannot use aggregate functions
  • HAVING filters groups AFTER GROUP BY — can use aggregates

Example

-- Find departments with more than 10 employees earning > 50000
SELECT department, COUNT(*) as emp_count
FROM employees
WHERE salary > 50000         -- filter rows first
GROUP BY department
HAVING COUNT(*) > 10;        -- filter groups after
💡 Interview Tip

Remember the execution order: WHERE runs before GROUP BY, HAVING runs after. So WHERE is more efficient — filter as early as possible.

EasyFind the second highest salary in a table.
▼

Multiple Approaches

-- Method 1: Subquery (works everywhere)
SELECT MAX(salary) FROM employees
WHERE salary < (SELECT MAX(salary) FROM employees);

-- Method 2: LIMIT OFFSET
SELECT DISTINCT salary FROM employees
ORDER BY salary DESC LIMIT 1 OFFSET 1;

-- Method 3: Window Function (best for Nth salary)
SELECT salary FROM (
  SELECT salary,
    DENSE_RANK() OVER (ORDER BY salary DESC) as rnk
  FROM employees
) ranked
WHERE rnk = 2;
💡 Interview Tip

Always show Method 3 with DENSE_RANK — it generalizes to "Nth highest salary" and shows you know window functions. Interviewers love that.

MediumWhat are Window Functions? Explain ROW_NUMBER, RANK, DENSE_RANK.
▼

Best Answer

Window functions perform calculations across a set of rows related to the current row, WITHOUT collapsing rows like GROUP BY does.

ROW_NUMBER vs RANK vs DENSE_RANK

-- Scores: 100, 100, 90, 80
-- ROW_NUMBER:  1, 2, 3, 4  (always unique)
-- RANK:        1, 1, 3, 4  (skips after tie)
-- DENSE_RANK:  1, 1, 2, 3  (no gaps)

SELECT name, salary,
  ROW_NUMBER() OVER (PARTITION BY dept ORDER BY salary DESC) as row_num,
  RANK()       OVER (PARTITION BY dept ORDER BY salary DESC) as rank,
  DENSE_RANK() OVER (PARTITION BY dept ORDER BY salary DESC) as dense_rank,
  LAG(salary)  OVER (PARTITION BY dept ORDER BY salary DESC) as prev_salary,
  SUM(salary)  OVER (PARTITION BY dept) as dept_total
FROM employees;
💡 Interview Tip

Window functions = OVER() clause. PARTITION BY = like GROUP BY but keeps all rows. Common uses: running totals, moving averages, percentiles, comparing row to group.

MediumWhat is an index? When should you and shouldn't you use one?
▼

Best Answer

An index is a separate data structure (usually B-tree or Hash) that speeds up data retrieval at the cost of additional storage and slower writes.

Use indexes on:

  • Primary keys (auto-indexed)
  • Foreign keys (JOIN columns)
  • Columns frequently in WHERE, ORDER BY, GROUP BY
  • High cardinality columns (many unique values)

Avoid indexes on:

  • Small tables (full scan faster)
  • Columns with few unique values (e.g., boolean)
  • Tables with heavy INSERT/UPDATE/DELETE (index maintenance cost)
💡 Interview Tip

Mention Composite Index column order matters — (dept, salary) helps queries filtering on dept but not queries filtering only on salary (leftmost prefix rule).

MediumExplain database normalization — 1NF, 2NF, 3NF.
▼

Best Answer

  • 1NF: Each column has atomic (indivisible) values. No repeating groups. Each row uniquely identified.
  • 2NF: 1NF + No partial dependencies. Every non-key attribute depends on the WHOLE primary key (matters for composite keys).
  • 3NF: 2NF + No transitive dependencies. Non-key attributes don't depend on other non-key attributes.
  • BCNF: Stronger 3NF — every determinant is a candidate key.

Simple Rule

"Every non-key attribute must depend on the key, the whole key, and nothing but the key."

💡 Interview Tip

Don't just define — give an example of a violation and how you'd fix it. E.g., storing CustomerCity in Orders table violates 3NF (City depends on CustomerID, not OrderID).

MediumWhat are ACID properties in databases?
▼

Best Answer

  • Atomicity: Transaction is all-or-nothing. Either all operations succeed or all are rolled back.
  • Consistency: Database moves from one valid state to another. All constraints are satisfied.
  • Isolation: Concurrent transactions don't interfere. Each transaction sees a consistent snapshot.
  • Durability: Committed transactions survive system failures (written to disk).

Isolation Levels (weakest to strongest):

  • Read Uncommitted → Read Committed → Repeatable Read → Serializable
💡 Interview Tip

Give a real example: "In a bank transfer, if debit succeeds but credit fails, Atomicity ensures the debit is rolled back." Concrete examples show you understand implications.

HardSQL: Find employees who earn more than their manager.
▼

Best Answer — Self Join

-- Table: Employee(id, name, salary, managerId)
SELECT e.name AS employee_name, e.salary,
       m.name AS manager_name, m.salary AS manager_salary
FROM Employee e
JOIN Employee m ON e.managerId = m.id
WHERE e.salary > m.salary;
💡 Interview Tip

This classic problem tests SELF JOIN. Always explain: "We treat the same table as two separate entities — one for the employee, one for the manager."

HardSQL vs NoSQL — when would you choose each?
▼

Choose SQL (Relational) when:

  • Data is structured with clear relationships
  • ACID compliance is critical (banking, healthcare)
  • Complex queries and reporting needed
  • Data integrity is paramount

Choose NoSQL when:

  • Massive scale and horizontal scaling needed
  • Flexible/evolving schema (document store like MongoDB)
  • High write throughput (Cassandra)
  • Key-value caching (Redis)
  • Graph relationships (Neo4j)

CAP Theorem:

Distributed systems can only guarantee 2 of 3: Consistency, Availability, Partition Tolerance. SQL → CA. Cassandra → AP. MongoDB → CP.

💡 Interview Tip

Don't say "NoSQL is better/faster." Instead: "It depends on the use case." Then demonstrate knowledge of CAP theorem and specific NoSQL types (document, key-value, column, graph).

MediumWrite a query to find duplicate emails in a table.
▼

Best Answer

-- Method 1: GROUP BY + HAVING
SELECT email, COUNT(*) as count
FROM Person
GROUP BY email
HAVING COUNT(*) > 1;

-- Method 2: Delete duplicates, keep one
DELETE FROM Person
WHERE id NOT IN (
  SELECT MIN(id)
  FROM Person
  GROUP BY email
);

-- Method 3: Window function approach
SELECT DISTINCT email FROM (
  SELECT email,
    ROW_NUMBER() OVER (PARTITION BY email ORDER BY id) as rn
  FROM Person
) t WHERE rn > 1;
💡 Interview Tip

If asked to DELETE duplicates, always clarify: "Which duplicate to keep — first inserted (MIN id) or last?" This shows attention to requirements.
